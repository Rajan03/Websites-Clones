Points to Remember

  1. JavaScript is case sensitive language means Time and TIME are two different keywords having different meaning .

  2. All white spaces and line breaks are ignored.

  3. Semicolons are optional, but they are recommended for code in one line

  4. JavaScript code is placed in Head tag, Body tag (recommended) using script tag in same file or as an external file with .js extension



Comments in JavaScript

  • Single line Comments are written using ' // ' whereas multiple line comments are written in between /* comment */.

Arrays- Non-Primitive datatype

  1. Object provides no method to manage the order of elements. so we use arrays for arrangements of elements in order.

  2. An array can store elements of any type.

  3. There are two syntaxes for creating an empty array:
    • let arr = new Array();
    • let arr = [];

  4. We can get an element by its number in square brackets.
    • alert( fruits[2] ); // Gives 3rd element of arrays.

  5. We can replace an element Or add a new one to the array:
    • fruits[3] = 'Lemon'; //replaces or adds new element to the array.

  6. The total count of the elements in the array is its length:
    • alert( arr.length );

Methods that work with the end of the array:
  • 1.pop
    Extracts the last element of the array and returns it.
    let fruits = ["Apple", "Orange"]; alert( fruits.pop() ); // remove "Orange" and alert it

  • 2.push
    Append the element to the end of the array: fruits.push("Pear"); //adds pear to array

        if a is defined, then a,
        if a isn’t defined, then b.

    Variables

    1. Variables are declared usingvar, let, and const keywords.
    2. Variables can't be a reserved keyword and should not start from a reserved keyword.

    3. The name must contain only letters, digits, or the symbols $ and _.

    4. Major differences in three keywords are:
      Variable Description
      var Syntax = var variableName=" value ";  or  var variableName;
      Has function or local scope only.
      Can be reassined but not declare same variable again.
      Can be initailised later.
      let Syntax = let variableName=" value ";   or  let variableName;
      Has global scope only.
      Can be reassined but not declare same variable again.
      Can be initailised later.
      const Syntax = const variableName=" value ";   or   const variableName;
      Has global scope only.
      Cannot be reassined but not declare same variable again.
      Can be initailised later.

    5. Modify-in-place: We often need to apply an operator to a variable and store the new result in that same variable.
      For example:
      let n = 2;
      n = n + 5;
      n = n * 2;
    6. Basic notations used for naming of variables or any fuction:
      • Camel case: first letter of second word is capital. e.g., fullName (recommended)

      • Pascal case: first letter of each word is capital. e.g., FullName

      • Underscore: words are differentiated using underscore'_' e.g., full_name

      • Uppercase constants: There is a widespread practice to use constants as aliases for difficult-to-remember values that are known prior to execution. Such constants are named using capital letters and underscores.

    If-Else

    The if(...) statement evaluates a condition in parentheses and, if the result is true, executes a block of code. if the result is false, else block is executed.

    The flowchart for if-else statement is:

    • filter_dramaSyntax
      if (expression){
       Statement(s) to be executed if expression is true
      }else{
       Statement(s) to be executed if expression is false
      }

    The ‘if...else if...’ statement is an advanced form of if…else that allows JavaScript to make a correct decision out of several conditions.

    • filter_dramaSyntax
      if (expression 1){
      Statement(s) to be executed if expression 1 is true
      }else if (expression 2){
      Statement(s) to be executed if expression 2 is true
      }else if (expression 3){
      Statement(s) to be executed if expression 3 is true
      }else{
      Statement(s) to be executed if no expression is true
      }


    Switch

    Nothing special just a replacement of If-else-If Statement. The objective of a switch statement is to give an expression to evaluate and several different statements to execute based on the value of the expression. The interpreter checks each case against the value of the expression until a match is found. If nothing matches, a default condition will be used.

    • filter_dramaSyntax
      switch (expression)
      {
      case condition 1: statement(s)
      break;
      case condition 2: statement(s)
      break;
      ...
      case condition n: statement(s)
      break;
      default: statement(s)
      }


    While Loop

    The purpose of a while loop is to execute a statement or code block repeatedly as long as an expression is true. Once the expression becomes false, the loop terminates.
    A single execution of the loop body is called an iteration.

    The flowchart for While loop is:


    • filter_dramaSyntax
      while (expression){
      Statement(s) to be executed if expression is true
      Increment/Decrement (if required)
      }


    Do-While Loop

    The do...while loop is similar to the while loop except that the condition check happens at the end of the loop. This means that the loop will always be executed at least once, even if the condition is false.


    • filter_dramaSyntax
      do{
      Statement(s) to be executed;
      Increment/Decrement (if required)
      } while (expression);


    For Loop

    For loop in leyman langugage means for given condition execute following statements. It includes the following three important parts:

    The flowchart for For Loop is:


    1. The loop initialization where we initialize our counter to a starting value. The initialization statement is executed before the loop begins.
    2. The test statement which will test if a given condition is true or not. If the condition is true, then the code given inside the loop will be executed, otherwise the control will come out of the loop.
    3. The iteration statement where you can increase or decrease your counter.


    • filter_dramaSyntax
      for (initialization; test condition; iteration statement){
      Statement(s) to be executed if test condition is true
      }


    For-in Loop

    The for...in loop is used to loop through an object's properties.
    In each iteration, one property from object is assigned to variablename and this loop continues till all the properties of the object are exhausted.


    • filter_dramaSyntax
      for (variablename in object){
      statement or block to execute
      }


    Loop Control

    Two types of statements are used to control flow of loop:


    • filter_dramaBreak Statement
      The break statement (break; ), is used to exit a loop early, breaking out of the enclosing curly braces.

    • filter_dramaContinuos Statement
      The continue statement (continue; ) tells the interpreter to immediately start the next iteration of the loop and skip the remaining code block.

    Label:
    A label is simply an identifier followed by a colon (:) that is applied to a statement or a block of code. The break labelName; statement in the loop breaks out to the label. A label can be used with break and continue to control the flow more precisely.

    1. Functions are used to perform a similar action again and again. The main purposes of functions is to avoid code duplication and follow DRY (Don't Repeat Youself) principle.


    2. Parameters Arguments
      Parameters are variables passed in function declarations. Arguments are the vaues given to parameters while calling a function.
      Any type of and any no. of parameters can be defined in declaration. No. of arguments can be greater or less than No. of parameters defined.
      1. If No.of arguments > No. of parameters, then arguments object method and Rest parameter can be used to access those extra arguments inside the functions.
      2. If No. of arguments < No.of parameters, then undefined value will be assined to extra parameters if no default value exists.
      Parameters defined while declaring a function can be used in function block only.
      Parameters can have a default value. but parameters with default value should be declared at last in parenthesis.


    3. In Declaration,
      The function keyword goes first, then name of function, then parameters.
      Function nameofFunction (Parameters){
      statements to be executed;
      return statement; (if requierd)
      }
      In Calling.
      To call a function, we use function name with paranthesis and semicolon at last. Also, function statements are executed where it is called not where it is declared.
      functionName();


    4. Argument Object Rest Parameter
      The arguments passed while calling a function is stored in Argument object which is object but not an array. Rest Parameter is an array that stores multiple arguments as array elements.
      Arguments object has access to argument.length property to find no. of arguments stored in the object. It is defined in parenthesis with other parameters or alone. It's syntax is like (a, ...anyName) where a and b are normal parameters and ...anyName is rest parameter.
      Arguments object count starts from 0. for example, In above syntax first argument will be assigned to 'a' parameter and all other arguments will be stored in rest parameter as an array.
      Arguments Object can be accessed using square brackets like arguments[0]. It must be the last parameter to the function, and, here also single element can be accessed using square brackets[ ].
